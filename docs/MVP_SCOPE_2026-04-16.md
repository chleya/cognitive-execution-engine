# MVP Scope

## MVP Definition

The MVP is not an agent demo.

The MVP is a working cognitive execution engine that proves:

- explicit state is more reliable than prompt-only continuity
- general cognitive primitives are more reusable than hard-coded templates
- policy and audit can control an LLM-assisted workflow

## In Scope

- fixed core state trunk
- typed event schema
- deterministic reducer
- policy decision model
- approval-required decision path
- audit log shape
- replay test
- bounded cognitive primitive contract
- deterministic planner
- constrained LLM task compiler adapter
- read-only tool proposal and observation flow

## Out of Scope

- multi-agent orchestration
- long-term autonomous memory learning
- self-modifying core policy
- open web autonomy
- tool marketplace
- persona simulation
- open-domain "general intelligence" claims

## MVP Domain Targets

First priority domains:

1. multi-step document analysis with traceable conclusions
2. rule-constrained workflow execution
3. cross-session continuous task management

## Acceptance Tests

- a malformed event is rejected
- a forbidden tool call is blocked
- a high-risk tool call returns `requires_approval`
- an approved state patch increments version exactly once
- replay of the event stream reconstructs the same state
- LLM output cannot directly become patch, tool call, or canonical belief

## Migration Test

The MVP is only credible if a second domain can be introduced by changing
plugins and evaluation logic rather than rewriting the core reducer/policy/audit
chain.
